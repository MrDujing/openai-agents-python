# Briefing Writer

Use this skill when the user asks for a concise project or operations briefing.

## Workflow

1. Identify the audience, deadline, and decision needed.
2. Extract only concrete facts, dates, blockers, owners, and next actions.
3. Produce a short briefing with these sections:
   - Situation
   - Risks
   - Next actions

## Constraints

- Keep the briefing under 200 words unless the user asks for more.
- Do not invent owners, dates, or metrics.
- If key context is missing, list the missing information at the end.
